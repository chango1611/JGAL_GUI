Metadata

La biblioteca JGAL permite implementar nuevas clases, con la finalidad de poder desarrollar AG con diferentes configuraciones a las que se encuentran por defecto. Esta herramienta permite agregar dichas clases a través del archivo metadata.file

Las nuevas clases que se vayan a agregar debe estar ya compiladas (.class) y deben estar ubicadas en una carpeta que se encuentre dentro del classpath (Ej. La carpeta donde se encuentra la aplicación)

Archivo Metadata.file

El archivo de metadatas sigue el siguiente formato:

  • <Metadata File> ::= <Genes Metadata> "\n" <Selectores Metadata> "\n" <Operadores Metadata> "\n" <Inicializadores Metadata>

  • <Genes Metadata> ::= Genes: "\n" <Cantidad Genes> {"\n" <Metadata>}

  • <Selectores Metadata> ::= Selectores: "\n" <Cantidad Selectores> {"\n" <Metadata>}

  • <Operadores Metadata> ::= Operadores: "\n" <Cantidad Operadores> {"\n" <Metadata>}

  • <Inicializadores Metadata> ::= Inicializadores: "\n" <Cantidad Operadores> {"\n" <Class Name> "\n" <Name>}

  • <Metadata> ::= <Class Name> "\n" <Name> "\n" <Cantidad Parametros> {"\n" <Parametros>}

  • <Parametros> ::= <Param Name> "\n" <Param Class>

Como se observa en el ejemplo, a los genes no se les coloca el nombre como parámetro (Éste debe estar en el constructor de la clase de todas manera). Lo mismo ocurre con los operadores, donde no se coloca la probabilidad de ocurrencia (Éste debe estar en el constructor de todas manera).

Nota 1. Las clases para los parámetros aceptados son:

  • java.lang.Integer

  • java.lang.Long

  • java.lang.Short

  • java.lang.Byte

  • java.lang.Double

  • java.lang.Float

  • java.lang.String

  • java.lang.Character

Nota 2. Las clases pasadas por la metadata deben tener metodos consultores(get) para todos los parámetros del constructor.
