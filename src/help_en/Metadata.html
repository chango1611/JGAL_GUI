Metadata

The library allows to implement new classes JGAL, in order to develop GA with different configurations than the default ones. This tool allows you to add these file types through the metadata.file

The new classes to be added must be already compiled (.class) and must be located in a folder that is within the classpath (eg. The folder where the application .jar is)

Archivo Metadata.file

The metadata file follows this format:

  • <Metadata File> ::= <Genes Metadata> "\n" <Selectors Metadata> "\n" <Operators Metadata> "\n" <Initializers Metadata>

  • <Genes Metadata> ::= Genes: "\n" <Number of Genes> {"\n" <Metadata>}

  • <Selectors Metadata> ::= Selectors: "\n" <Number of Selectors> {"\n" <Metadata>}

  • <Operators Metadata> ::= Operators: "\n" <Number of Operators> {"\n" <Metadata>}

  • < Metadata> ::= Inizializadores: "\n" <Number of Operators> {"\n" <Class Name> "\n" <Name>}

  • <Metadata> ::= <Class Name> "\n" <Name> "\n" <Number of Parameters> {"\n" <Parameters>}

  • <Parameters> ::= <Param Name> "\n" <Param Class>

As shown in the example, the genes has not the name as a parameter (This should be in the class constructor anyway). The same happens with the Operators probability of occurrence (It should be in the constructor anyway).

Note 1. The classes for the accepted parameters are:

  • java.lang.Integer

  • java.lang.Long

  • java.lang.Short

  • java.lang.Byte

  • java.lang.Double

  • java.lang.Float

  • java.lang.String

  • java.lang.Character

Note 2. The classes in the metadata must have methods get for all the parameters described in the constructor.
