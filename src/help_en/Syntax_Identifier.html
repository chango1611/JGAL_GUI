Identifiers

Identifiers allow constant values ??or variables be used in operations inside the function.

  • <Identifier> ::= $(<Name> | "{"<Arithmetic Statement>"}")

There are two ways to access an identifier, by name or by position.

By name it search in the list of identifiers and returns the variable with that name.

By position the identifier is selected by the position that is passed. If the position is greater than the amount of current variables -1, then the parser will throw an error. GAL_Parser considered the first positions (0 .. N-1), as the positions of the N constant values.

e.g:

//Assuming that there are three constant values
$i:= 4;
$j:= 5;
${3}+$j;

The above code assigns the value 4 to the variable i and 5 to the variable j.
After this, an operation is performed between two identifiers ${3} and $j.
${3} is an identifier by position, so that the identifier that is returned is in position 3, which would be $i, since the position from 0 to 2 are the constant values ??and the first variable declared was $i which is equal to 4.
The second identifier is by name, it is achieved that $j is equal to 5.
It returns the result of the sum, which would be 4+5=9.

Return

Identifiers return the most recent value of the variable or constant.
